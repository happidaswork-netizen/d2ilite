# 加名字白条功能规格（2026-06-26）

## 1. 功能目标

给当前图片底部追加一条白色横条，并在横条上写入角色名/人物名，生成新的派生图片。

典型效果：

```text
┌─────────────┐
│             │
│    原图      │
│             │
├─────────────┤
│    张三      │
└─────────────┘
```

该功能属于 D2I Lite 的“当前图片动作”，不应作为独立脚本长期维护。

## 2. 入口

### GUI

在 D2I Lite 打开一张图后：

```text
当前图片工具 -> 加名字白条
```

弹出小设置框：

- 角色名
- 输出格式
- 命名方式
- 是否打开输出文件夹

### 命令行 / Quicker / 右键

建议命令：

```powershell
python app.py --action name-bar "D:\images\001.jpg"
python app.py --action name-bar "D:\images\001.jpg" --name "张三"
python app.py --action name-bar "D:\images\001.jpg" --name "张三" --format png --output-name label
```

打包后：

```powershell
D2ILite.exe --action name-bar "D:\images\001.jpg" --name "张三"
```

Quicker 只负责把“当前选中的文件路径”作为参数传进来，不允许脚本去扫描资源管理器窗口猜路径。

## 3. 角色名来源优先级

自动取名时按以下顺序：

1. `ImageMetadataInfo.person`
2. XMP `Xmp.iptcExt.PersonInImage`
3. `Xmp.titi.meta.d2i_profile.name`
4. `ImageMetadataInfo.title`，但要避免把“姓名 - 简介”整体当成姓名
5. 文件名 stem
6. 用户手动输入

若无法确认，必须弹窗让用户确认，不要静默使用错误名字。

## 4. 输出格式

默认：`PNG`。

原因：PNG 是无损编码，可以避免 JPG 二次压缩造成画质损失。

可选：

| 格式 | 用途 | 说明 |
| --- | --- | --- |
| PNG | 默认、长期保存、素材归档 | 无损，文件较大 |
| JPG | 需要小体积或兼容旧平台 | 必然重新编码，不是严格无损；默认质量建议 100，4:4:4 subsampling |
| WEBP | 可选高级格式 | 默认无损或高质量 |
| 保持原格式 | 保持用户习惯 | 原图是 JPG 时仍会重新编码 |

## 5. 输出命名

必须支持两种命名方式：

### A. 原图名 + 后缀

```text
IMG_6303.jpg -> IMG_6303_named.png
```

默认后缀：`_named`。

### B. 直接用角色名命名

```text
IMG_6303.jpg + 张三 -> 张三.png
```

这是用户明确需要的模式，应作为配置项：

```text
output_name_mode = label
```

重名处理：

```text
张三.png
张三_2.png
张三_3.png
```

必须做文件名清洗：替换 Windows/macOS/Linux 不安全字符，如：

```text
< > : " / \ | ? * 控制字符
```

不得覆盖原图。即使开启 overwrite，也不得把输出路径解析为当前原图路径。

## 6. 白条与文字样式

建议默认值：

```text
bar_color = white
text_color = black
bar_height = 原图高度 * 0.14，最低 48px
align = center
font = 自动选择中文字体
```

字体自动选择顺序：

- Windows：微软雅黑/黑体/等线
- macOS：PingFang/STHeiti
- Linux：Noto Sans CJK/DejaVu Sans

文字必须自动缩放，保证长名字不会溢出白条。

## 7. 图片处理要求

- 使用 Pillow。
- 打开图片后先做 EXIF orientation 修正。
- 保留 ICC profile，尽量不破坏色彩管理。
- PNG/WebP 透明图需要先按白色背景合成，再追加白条。
- 输出派生图，不修改原图。
- 成功后返回 `output_path`，GUI/Quicker 可打开所在文件夹并选中。

## 8. 建议代码位置

```text
actions/name_bar.py
services/image_rendering.py
```

短期最小实现可放：

```text
image_name_bar.py
```

但不要把核心渲染逻辑写在 GUI 回调里。

## 9. 验收用例

1. JPG 输入，默认输出 PNG，尺寸高度增加，原图不变。
2. PNG 输入，名字为中文，文字不乱码。
3. 角色名很长时自动缩小字号。
4. 输出命名 `label` 时生成 `张三.png`。
5. `张三.png` 已存在时生成 `张三_2.png`。
6. 原图文件名包含空格/中文时路径正常。
7. Quicker 调用传入路径时不选错图片。
8. JPG 输出质量设为 100 时使用 4:4:4，文档提示仍非严格无损。
