# AI 单图动作接口规划（2026-06-26）

## 1. 定位

D2I Lite 的 AI 能力定位为“当前图片的随手动作”。

它和 TITI 的区别：

| 能力 | D2I Lite | TITI |
| --- | --- | --- |
| 使用场景 | 当前图片、单次处理、人工确认 | 批量生成、批量编排、生产流水线 |
| 输入 | 当前图片 + 元数据 + 用户少量选项 | 任务队列、参考图组、提示词编排 |
| 输出 | 单个或少量派生文件 | 成批作品与归档 |
| 入口 | GUI 按钮、右键、Quicker | TITI 工作台 |

D2I Lite 不应复制 TITI 的批量生成系统，只提供单图入口和轻量动作。

## 2. 第一批动作

建议先做以下动作：

| 动作 ID | 名称 | 输入 | 输出 |
| --- | --- | --- | --- |
| `name-bar` | 加名字白条 | 当前图片 + 名字 | 派生图片 |
| `ai-repair` | AI 修复图片 | 当前图片 | 修复后的派生图片 |
| `ai-generate` | AI 单图生图 | 当前图片 + 文字要求 | 新图片 |
| `ai-prompt` | 反推提示词/描述 | 当前图片 | 文本，可写入元数据 |
| `metadata-fill` | AI 补全元数据 | 当前图片 + 现有字段 | 元数据草稿，用户确认后保存 |

## 3. 模型网关设计

新增统一模型网关，不要在每个按钮里直接写 API 调用。

建议文件：

```text
services/model_gateway.py
```

接口草案：

```python
@dataclass
class ModelRequest:
    action: str
    image_path: str | None = None
    prompt: str = ""
    metadata: dict | None = None
    options: dict | None = None

@dataclass
class ModelResponse:
    ok: bool
    message: str
    text: str = ""
    output_paths: list[str] = field(default_factory=list)
    raw: dict | None = None

class ModelGateway:
    def run(self, request: ModelRequest) -> ModelResponse: ...
```

Provider 配置必须可替换：

```text
openai-compatible
local-webui
custom-command
```

短期推荐先做 `openai-compatible`，因为能兼容很多本地或远程模型代理。

## 4. 配置与密钥

不要把 API Key 写进仓库。

建议配置：

```json
{
  "model_gateway": {
    "provider": "openai-compatible",
    "base_url": "http://127.0.0.1:8000/v1",
    "api_key_env": "D2I_MODEL_API_KEY",
    "image_model": "",
    "vision_model": ""
  }
}
```

本地 GUI 可以提供设置入口，但保存时应只保存环境变量名或本地用户配置。若必须保存密钥，也应放在用户目录的 local config，不进 git。

## 5. 输出原则

AI 动作默认生成派生文件，不覆盖原图。

建议输出目录：

```text
原图同目录/_d2i_actions/
```

或用户配置的输出目录。

命名：

```text
原图stem_ai_repair.png
原图stem_ai_generate_时间戳.png
```

对于会修改当前图片的动作，必须做二次确认，并先备份原图。

## 6. GUI 交互原则

单图动作不做复杂工作流。建议每个动作弹一个轻量对话框：

- 当前图片缩略信息
- 动作选项
- 输出格式/目录
- 执行按钮
- 输出结果路径

AI 结果必须可回看：

- 成功后打开/定位输出文件
- 错误时显示 provider、请求目标、简短错误原因
- 详细日志写入本地日志文件

## 7. Quicker / 右键入口

命令格式：

```powershell
D2ILite.exe --action ai-repair "D:\images\sample.jpg"
D2ILite.exe --action ai-generate "D:\images\sample.jpg"
```

Quicker 只插入当前选中文件路径。不要再使用脚本扫描资源管理器窗口。

## 8. 和现有能力的关系

现有 D2I Lite 已有：

- `AI自动补全`
- `AI自动小传`
- `下载修复`
- `直连修复`

后续应把这些能力逐步包装成动作，而不是保留两套入口逻辑。

优先级建议：

1. `name-bar`：无外部依赖，先打通动作系统。
2. `metadata-fill`：复用现有 AI 元数据补全，验证模型网关。
3. `ai-repair`：单图输入/输出，输出派生图。
4. `ai-generate`：与 TITI 保持边界，只做单图快捷生成。

## 9. 验收口径

- 未配置模型时，AI 动作按钮能提示配置缺失，不崩溃。
- 配置 provider 后，单图动作可执行，并输出派生文件或文本结果。
- 错误日志可定位，不把完整密钥打印出来。
- Quicker 调用和 GUI 调用走同一个动作入口。
- AI 动作不污染原图元数据，除非用户明确点击保存。
